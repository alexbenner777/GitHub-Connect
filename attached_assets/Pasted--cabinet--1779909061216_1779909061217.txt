БЛОК 1 — БЕЗОПАСНОСТЬ JWT (критично)
═══════════════════════════════════════════

1.1. Переведи JWT-аутентификацию с localStorage на httpOnly cookies:
    - На бэке: при успешном /login установи cookie с флагами:
      res.cookie('trends_jwt', token, {
        httpOnly: true,
        secure: true,
        sameSite: 'strict',
        maxAge: 30 * 24 * 60 * 60 * 1000,
        path: '/'
      })
    - На фронте: убери все обращения к localStorage.getItem('trends_jwt') 
      и localStorage.setItem('trends_jwt', ...). Cookie уезжает с запросами автоматически.
    - На бэке: проверь auth-middleware, чтобы JWT читался из req.cookies, а не из Authorization header.
    - Добавь cookie-parser к Express, если ещё не подключён.

1.2. Из JWT payload убери email. Оставь только:
    { userId, isAdmin, iat, exp }
    Email подтягивай отдельным запросом /api/me на бэке по userId.

1.3. Добавь CSRF-защиту:
    - Сгенерируй CSRF-токен при логине и положи его в обычную (не httpOnly) cookie.
    - На фронте отправляй его в заголовке X-CSRF-Token при всех POST/PUT/DELETE.
    - На бэке валидируй (double-submit pattern).

1.4. Сократи срок жизни access JWT до 60 минут. Добавь refresh-токен:
    - Refresh JWT с exp 7 дней, тоже в httpOnly cookie.
    - Endpoint POST /api/auth/refresh выдаёт новый access token.
    - При ротации старый refresh инвалидируется.

БЛОК 2 — БЕЗОПАСНОСТЬ КОШЕЛЬКА (критично)
═══════════════════════════════════════════

2.1. Смена адреса кошелька в Настройках должна требовать подтверждение:
    - При нажатии "Сохранить кошелёк" отправь на email пользователя 6-значный код.
    - Пользователь вводит код во всплывающем окне.
    - Только после успешной верификации запиши новый адрес в БД.
    - Логируй все изменения адреса в таблицу wallet_changes (userId, oldAddress, newAddress, ip, userAgent, timestamp).

2.2. Валидация адресов по сети:
    - TON: проверка regexp ^[UE]Q[A-Za-z0-9_-]{46}$
    - USDT TRC-20: ^T[A-Za-z1-9]{33}$
    - USDT ERC-20: ^0x[a-fA-F0-9]{40}$ + EIP-55 checksum
    - BTC: bech32 или base58check валидация
    Если адрес не проходит — не давай сохранить.

2.3. Под селектором сети добавь подсказку: 
    "В этой сети вы получаете выплаты в USDT" (или соответствующей монете).
    Для каждой сети — отдельный текст.
